# Threat Modeling (STRIDE / PASTA) — Junior

1. What is threat modeling? — Identifying threats and mitigations early.
2. What is STRIDE? — Spoofing, Tampering, Repudiation, Info disclosure, DoS, Elevation.
3. What is PASTA? — A risk-centric threat modeling methodology.
4. What is a data flow diagram (DFD)? — Visual of data movement and trust boundaries.
5. What is a trust boundary? — A point where trust changes.
6. What is an asset? — Anything valuable to protect.
7. What is a threat actor? — Entity that can cause harm.
8. What is a mitigation? — A control that reduces risk.
9. What is an abuse case? — A misuse scenario.
10. What is residual risk? — Risk that remains after controls.
11. What is impact vs likelihood? — Severity vs probability.
12. What is attack surface? — Exposed entry points.
13. What is a threat model scope? — System boundaries included.
14. What is a security requirement? — A measurable control.
15. Why do threat models need updates? — Systems and threats change.
