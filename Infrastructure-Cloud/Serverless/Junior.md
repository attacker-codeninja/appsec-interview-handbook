# Serverless Security — Junior

1. What is serverless? — Managed compute like AWS Lambda.
2. What is a trigger? — Event source for functions.
3. Why is IAM important in serverless? — Controls function permissions.
4. What is event injection? — Malicious data sent to triggers.
5. What is the shared responsibility model? — Cloud provider vs customer responsibilities.
6. What is a function timeout? — Max execution time.
7. What is a dead-letter queue? — Stores failed events.
8. What is an API Gateway? — Front door for serverless APIs.
9. What is a serverless cold start? — First invocation latency.
10. What is environment variable risk? — Secrets can leak.
11. What is log retention? — Keeping logs for incident response.
12. What is least privilege? — Minimal permissions needed.
13. What is dependency pinning? — Fixing library versions.
14. What is function versioning? — Immutable function versions.
15. What is a serverless workflow? — Orchestration of functions.
