# Secure SDLC — Junior

1. What is Secure SDLC? — Integrating security into all phases of development.
2. Why do we shift left? — Fixing early is cheaper and safer.
3. What is a security requirement? — A specific control to reduce risk.
4. What is a security gate? — A checkpoint before release.
5. What is a security champion? — Developer who supports security practices.
6. Why do code reviews matter? — Catch issues and enforce standards.
7. What is a vulnerability backlog? — Tracked security issues for remediation.
8. What is a security baseline? — Minimum required controls.
9. What is secure coding? — Practices that reduce vulnerabilities.
10. What is a security policy? — Organizational rules for security.
11. Why are dependencies part of SDLC? — They introduce risk.
12. What is a penetration test? — Manual security assessment.
13. What is a bug bounty? — External reporting of vulnerabilities.
14. What is a security SLA? — Time-to-fix targets.
15. What is security monitoring? — Detection of attacks and anomalies.
