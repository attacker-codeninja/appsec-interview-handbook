# OWASP Top 10: Vulnerable and Outdated Components — Junior / Entry Level

1. What is a vulnerable component? — A library with known security issues.
2. What is a CVE? — Publicly disclosed vulnerability identifier.
3. What is SCA? — Software composition analysis.
4. What is a transitive dependency? — A dependency of a dependency.
5. Why are EOL frameworks risky? — No security patches.
6. What is dependency pinning? — Locking versions to known good releases.
7. What is a lockfile? — Captures exact dependency versions.
8. What is typosquatting? — Malicious packages with similar names.
9. Why are container base images important? — They include vulnerable packages.
10. What is SBOM? — Software bill of materials.
11. What is a supply-chain attack? — Compromising dependencies or build tools.
12. What is a malicious package? — A library with embedded malware.
13. Why are unused dependencies risky? — They expand attack surface.
14. What is a version constraint risk? — Allows vulnerable versions.
15. What is a vendored dependency? — Bundled third-party code.
