# OWASP Top 10: Security Misconfiguration — Junior / Entry Level

1. What is a security misconfiguration? — Unsafe default or incorrect settings.
2. Why is debug mode risky? — It exposes stack traces and internal info.
3. What is a default credential risk? — Known usernames/passwords.
4. What is directory listing risk? — Exposes files and structure.
5. What is a missing security header? — Increases XSS/clickjacking risk.
6. What is permissive CORS? — Allows untrusted origins to access APIs.
7. What is an open admin interface? — Unrestricted access to admin tools.
8. Why is unused services exposure risky? — Extra attack surface.
9. What is a public S3 bucket risk? — Data exposure.
10. What is a misconfigured firewall? — Unintended open ports.
11. What is verbose error leakage? — Exposes internal data.
12. Why is default cloud networking risky? — Broad access by default.
13. What is a missing TLS redirect? — Allows HTTP downgrade.
14. What is a weak CSP? — Allows unsafe script sources.
15. What is insecure file permission? — Excessive access to sensitive files.
