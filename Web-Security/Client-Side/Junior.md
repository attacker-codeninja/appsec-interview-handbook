# Client-Side Attacks — Junior

1. What is DOM-based XSS? — XSS that occurs entirely in the browser.
2. What is clickjacking? — Tricking users into clicking hidden UI.
3. What is a CSP and why is it useful? — Policy to restrict script sources.
4. What is an open redirect? — Redirecting users to untrusted sites.
5. What is subresource integrity (SRI)? — Hash-based verification for external scripts.
6. What is a mixed content warning? — Loading HTTP resources on HTTPS pages.
7. What is client-side template injection? — Injecting code into template engines.
8. What is localStorage risk? — Tokens can be stolen by XSS.
9. What is a browser extension risk? — Extensions can read and modify pages.
10. What is tabnabbing? — Changing the origin page via `window.opener`.
11. What is postMessage misuse? — Accepting messages without origin checks.
12. What is DOM clobbering? — Overriding DOM variables with elements.
13. Why are inline scripts risky? — Hard to control via CSP.
14. Why is input validation still needed on the server? — Client can be bypassed.
15. What is UI redress? — Visual trick to get unintended actions.
