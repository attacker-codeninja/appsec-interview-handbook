# OWASP Top 10: Security Logging and Monitoring Failures — Junior / Entry Level

1. Why are security logs important? — They enable detection and forensics.
2. What is an audit log? — A record of security-relevant actions.
3. What is log retention? — How long logs are stored.
4. What is log tampering? — Modifying logs to hide actions.
5. What is a SIEM? — Central system for log analysis and alerts.
6. What is a correlation ID? — Identifier linking events across services.
7. What is an alert? — Notification of suspicious activity.
8. What is PII in logs risk? — Sensitive data leakage.
9. What is log injection? — Manipulating log entries with crafted input.
10. What is a failed login log? — Record of unsuccessful auth attempts.
11. Why are timestamps important? — They support investigation timelines.
12. What is an incident response playbook? — Steps to handle incidents.
13. What is monitoring coverage? — Systems/events being observed.
14. What is log centralization? — Aggregating logs in one place.
15. Why are noisy logs bad? — They hide real attacks.
