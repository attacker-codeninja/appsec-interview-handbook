# OAuth2 / OpenID Connect Security — Junior

1. What is OAuth 2.0? — Authorization framework for delegated access.
2. What is OpenID Connect? — Identity layer on top of OAuth 2.0.
3. What is an access token? — A token that grants access to APIs.
4. What is a refresh token? — A token used to get new access tokens.
5. What is PKCE and why is it needed? — Protects public clients from code interception.
6. What is the `state` parameter? — Protects against CSRF in OAuth flows.
7. What is an ID token? — OIDC token with user identity claims.
8. What is the authorization code flow? — Server gets code then exchanges for token.
9. What is the client credentials flow? — Machine-to-machine auth without user context.
10. What is redirect URI validation? — Ensures tokens go to trusted endpoints.
11. What is token scope? — Limits permissions of a token.
12. What is a token audience (`aud`)? — Identifies intended recipient.
13. What is JWT? — A signed token format.
14. What is token expiration and why is it required? — Limits abuse window.
15. What is token revocation? — Invalidating tokens before expiration.
