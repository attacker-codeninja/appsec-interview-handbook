# CI/CD Security (SAST/DAST/SCA Integration) — Junior

1. What is SAST? — Static analysis of source code.
2. What is DAST? — Dynamic testing of running apps.
3. What is SCA? — Scanning dependencies for CVEs.
4. What is a CI pipeline? — Automated build/test steps.
5. Why scan for secrets? — Hardcoded credentials are high risk.
6. What is a build artifact? — Compiled output like binaries or images.
7. What is code signing? — Verifying artifact integrity.
8. What is least privilege in CI? — Minimal permissions for runners.
9. Why are build logs sensitive? — Can leak secrets and internal data.
10. What is a gated merge? — Blocking merge until checks pass.
11. What is a vulnerability threshold? — Score limit to fail builds.
12. Why is dependency pinning important? — Reduces supply-chain drift.
13. What is a pipeline secret? — Credentials used by CI.
14. What is a SBOM? — Software Bill of Materials.
15. What is a runner? — The machine executing CI jobs.
