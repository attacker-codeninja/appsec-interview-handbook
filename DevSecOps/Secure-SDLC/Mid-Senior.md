# Secure SDLC — Mid-Senior

1. How do you set security gates without blocking delivery? — Risk-based gating and exceptions.
2. How do you measure SDLC security maturity? — Metrics like fix time and coverage.
3. How do you prioritize security debt? — Risk scoring and business impact.
4. How do you implement security requirements at scale? — Templates and reusable controls.
5. How do you enforce secure coding standards? — Linters and code review checklists.
6. How do you integrate security testing into agile? — CI scans and sprint planning.
7. How do you manage vulnerability SLAs? — Triage, ownership, and tracking.
8. How do you handle third-party risk in SDLC? — Vendor assessments and contract clauses.
9. How do you educate developers effectively? — Targeted training and secure patterns.
10. How do you handle security exceptions? — Formal risk acceptance with expiration.
11. How do you align with compliance (SOC2/PCI)? — Map controls to requirements.
12. How do you handle security in feature flags? — Limit exposure and monitor.
13. How do you run a secure release process? — Signed artifacts and approvals.
14. How do you embed security in product teams? — Security champions and coaching.
15. How do you validate remediation? — Retesting and automated verification.
